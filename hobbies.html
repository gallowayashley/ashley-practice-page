CSS Notes (to be moved)

Note: By having inline CSS (the code within brackets in HTML aka no line breaks), internal CSS (writing style CSS element into HTML), and external CSS (CSS stylesheet) affecting the same property of the same element, the priority is given to the inline element. Then internal then external CSS and it's usually best practice as a web designer to implement all styles in external CSS stylesheet.

  1. Inline CSS priority
  2. Internal CSS priority
  3. External CSS priority
Always have:
selector [the "who's style do you want to change"? h1, h2, img tags, etc.]
{ property [the "what about?" the selector's style that you want to change? background color, text color, position, etc.]: value; [the "how?" ie how do you want to change the background color of h1?]}
and this is where the CSS rules reside. The rules will change the appearance of some property and give it a new value.
